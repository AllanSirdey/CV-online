Note technique

Ci dessous quelques détails technique relatifs à la création du projet "CV en ligne".
Le site a été développé avec l'éditeur Brackets.
Le code source est disponible sur mon espace github (https://github.com/AllanSirdey)

HTML

  • Le site est compatible xHTML5 (sauf pour les balises où Angular est utilisé).
  • Externaliser les CSS et JavaScript : règle 16 opquest éco-conception web
  • Fournir une CSS print : règle 30 opquest éco-conception web
  • Les arrières-plan sont combinés dans un sprite CSS : règle 6 opquest webperf
  • Le code HTML est minifié : règle 9 opquest webperf

CSS

  • Le site est responsive grâce à Bootstrap v3.
  • J'ai ajouté quelques règles en plus, notamment pour la feuille de style "print" pour l'impression.
  • Le design a été testé sur firefox 55 desktop, chrome 55 et internet explorer 11.
  • Le responsive a été testé sur chrome avec l'extension "Responsive Web Design Tester".
  • Utilisation des sprites css pour afficher les icônes des boutons sociaux.
  • La feuille de style principal "style.css" est validé par le validateur "W3C CSS Validation Service".
  • La feuille de style principal "style.css" est validé par CSSLINT avec 0 erreurs et 0 warnings (après correction de 7 warnings).
  • La feuille de style principal "style.css" a été minifié grâce au site "cssminifier.com".
  • Pour le design, je me suis inspiré du guide "Material Design" de Google, principalement pour les couleurs.

JAVASCRIPT

  • Utilisation de Jquery pour créer le menu responsive lorsque la largeur d'écran est réduite ( < 768px ).
  • Utilisation de Jquery et VanillaJS pour l'intégration du Konami Code.
  • Source: http://xumi.fr/articles/ajouter-un-konami-code-a-son-site
  • Utilisateur de Vanilla JS lors du clic sur bouton "Click me".
  • Utilisateur de Angular pour l'include de fichier html externe (menu.html, informations.html).
  • Utilisation d'Ajax pour faire un appel à l'API Google Maps.
  • Javascript non validé par JSlint.

AMÉLIORATIONS

  • Manque d'inspiration pour l'utilisation de VanillaJS ou encore Angular.
  • Création d'un backend pour ajouter dynamiquement de nouvelles données (compétences, expériences, etc...)
